JsPatternToolPlus 模块

提供图像模板匹配、模型创建和坐标仿射变换功能

JsPatternToolPlus 模块封装了基于Halcon的模板匹配工具，提供了从文件读取图像、创建模板匹配模型、执行模板匹配以及坐标仿射变换的功能。该模块支持高精度的图像匹配和位置识别，广泛应用于工业视觉定位、零件检测和质量控制场景。

核心 API 详解

void CreateModel(string str_Name_Machine)
为指定的机器/工具名称创建模板匹配模型 。改方法为C#内部接口调用
参数名 类型 描述 必填
str_Name_Machine string 唯一标识符，用于指定要创建或使用哪个模板匹配工具实例 是
返回值:

无返回值(void)，但会在 VisionHelper 中创建或获取指定的模板匹配工具

注意: 如果模型创建失败，错误将记录到系统日志中，但不会抛出异常
JsPatternToolPlus.Run(string str_Name_Machine, Bitmap bmp)
在指定的位图图像上执行模板匹配，并返回检测结果
参数名 类型 描述 必填
str_Name_Machine string 唯一标识符，用于指定要使用的模板匹配工具实例 是
bmp Bitmap System.Drawing.Bitmap 类型的输入图像，在此图像上执行模板匹配 是
返回值:

返回一个 Dictionary<string, object> 类型的对象，包含的键值对与 Run(string) 方法相同

键名 类型 描述 条件
"res" int 执行结果代码：0 表示成功，非0 表示失败 总是存在
"Row" double 匹配到的模板中心Y坐标（行坐标） 总是存在
"Column" double 匹配到的模板中心X坐标（列坐标） 总是存在
"Angle" double 匹配到的模板角度，单位为弧度 总是存在
"Score" double 匹配得分（相似度），范围通常为 0.0 ~ 1.0 总是存在
错误处理: 当 res 不为0时，错误代码会被记录到系统日志中，格式为："模板匹配失败：[str_Name_Machine] error:[res]"

完整使用示例

// 示例1: 基本模板匹配流程
// ====================================

// 读取待检测图像
const testImage = JsImage.ReadImage('./test/test_image.bmp');

const result = patternTool.Run('ChipDetector', testImage);

if(result.res === 0) {
  // 匹配成功
  JsLog.info('模板匹配成功!');
  JsLog.info('模板中心坐标 (X, Y): (' + result.Column.toFixed(2) + ', ' + result.Row.toFixed(2) + ')');
  JsLog.info('旋转角度: ' + result.Angle.toFixed(4) + ' 弧度');
  JsLog.info('匹配得分: ' + result.Score.toFixed(4));
  
  // 通常高于0.7的得分表示良好匹配
  if(result.Score > 0.7) {
    JsLog.info('匹配质量: 优秀');
  } else if(result.Score > 0.5) {
    JsLog.info('匹配质量: 良好');
  } else {
    JsLog.info('匹配质量: 较差，建议重新检查');
  }
} else {
  // 匹配失败
  console.error('模板匹配失败! 错误代码: ' + result.res);
}
