jsFindCircleToolPlus 模块

提供图像中圆形的读取、模型创建和检测功能

jsFindCircleToolPlus 模块封装了基于Halcon的圆形查找工具，提供了从文件读取图像、创建圆形查找模型以及在图像中执行圆形查找的功能。该模块支持高精度的圆形检测，适用于工业视觉检测场景。

核心 API 详解

jsFindCircleToolPlus.ReadImage(string ImagePaht)
从指定路径读取图像文件，并将其存储在内部变量中
参数名 类型 描述 必填
ImagePaht string 图像文件的完整路径，支持Halcon支持的所有图像格式 是
返回值:

无返回值(void)，图像被加载到内部 image 变量中用于后续操作

jsFindCircleToolPlus.CreateModel(string str_Name_Machine)
为指定的机器/工具名称创建圆形查找模型，属于C#内部程序调用
参数名 类型 描述 必填
str_Name_Machine string 唯一标识符，用于指定要创建或使用哪个圆形查找工具实例 是
返回值:

无返回值(void)，但会在 VisionHelper 中创建或获取指定的圆形查找工具

注意: 如果模型创建失败，错误将记录到系统日志中，但不会抛出异常
jsFindCircleToolPlus.Run(string str_Name_Machine, Bitmap bitmap)
在指定的位图图像上执行圆形查找，并返回检测结果
参数名 类型 描述 必填
str_Name_Machine string 唯一标识符，用于指定要使用的圆形查找工具实例 是
bitmap Bitmap System.Drawing.Bitmap 类型的输入图像，在此图像上执行圆形查找 是
返回值:

返回一个 Dictionary<string, object> 类型的对象，包含以下键值对：

键名 类型 描述 条件
"res" int 执行结果代码：0 表示成功，非0 表示失败 总是存在
"Row" double 检测到的圆心Y坐标（行坐标） 仅当 res=0 时存在
"Column" double 检测到的圆心X坐标（列坐标） 仅当 res=0 时存在
"Radius" double 检测到的圆的半径，单位为像素 仅当 res=0 时存在
错误处理: 当 res 不为0时，错误代码会被记录到系统日志中，格式为："找圆失败：[str_Name_Machine] error:[res]"

完整使用示例

// 读取待检测图像
const testImage = JsImage.ReadImage('./test/test_image.bmp');

// 执行圆形检测
const result = circleTool.Run('MyCircleDetector', testImage);

// 处理检测结果
if(result.res === 0) {
  // 检测成功
  JsLog.info('检测成功!');
  JsLog.info('圆心坐标 (X, Y): (' + result.Column.toFixed(2) + ', ' + result.Row.toFixed(2) + ')');
  JsLog.info('半径: ' + result.Radius.toFixed(2) + ' 像素');
  
  // 可以使用这些参数进行后续处理
} else {
  // 检测失败
  console.error('圆形检测失败! 错误代码: ' + result.res);
  
  // 可能的原因：
  // 1. 图像中没有符合要求的圆形
  // 2. 模型参数不适合当前图像
  // 3. 图像质量问题
  // 4. 指定的工具名称不正确
}

注意事项

  • 在调用 Run 方法前，必须先调用 CreateModel 方法创建模型
  • 同一个工具名称（str_Name_Machine）会共享同一个模型配置
  • 当 res 不为0时，返回的字典中不会包含 "Row"、"Column" 和 "Radius" 键
  • 所有错误信息都会记录到系统日志中，可以通过日志系统查看详细错误
  • 该模块依赖于 HalconDotNet 和 System.Drawing 命名空间
